# ipm-render

`ipm-intro` holds source markdown (with inline ` ```ipmt ` blocks) and `.ipmt` files under `docs/`. `ipm-render` is a thin orchestrator that refreshes the sibling repo [ipm-intro-rendered](https://github.com/infinite-pm/ipm-intro-rendered) with images and rendered markdown.

## Where it lives

`ipm-drawio/cmd/ipm-render/` — same Go module as the other rendering binaries, so it imports `pkg/ipmsvg`, `pkg/drawio/gen`, and the (new) shared block-extraction package directly instead of shelling out.

This stub doc stays in `ipm-intro/tools/ipm-render/` as the user-facing entry point that points at the implementation.

## How it works

The heavy lifting already exists in [ipm-drawio](../../../ipm-drawio/) — `ipm-render` just wires it up:

1. **Pre-flight** — verify `ipm-intro-rendered` is a clean working tree (`git status --porcelain` empty). If dirty, **abort** with a message pointing at the offending paths. This protects against wiping local edits.
2. **Wipe** `ipm-intro-rendered/docs/` (full clear → obsolete files are removed automatically). README and `.git/` at the rendered repo root are untouched.
3. **Walk** `ipm-intro/docs/` for `.md` and `.ipmt` files; mirror the relative path under `ipm-intro-rendered/docs/`.
4. **Render** — two SVG flavors per block/file:
   - **Native** (`.ipm.svg`) — `ipmsvg-gen --in <file.ipmt>` (accepts `.ipmt` directly). Pure-Go, no container, fast, minimal styling. Default and always produced.
   - **draw.io** (`.drawio.svg`) — `ipmt-parse | drawio-gen`. Styled, editable in draw.io / hediet's VS Code extension. Requires Podman + draw.io renderer container; opt-in via flag.
   - For `.md` files, block extraction is shared with `ipmt-inline-md` and `debug-layout-md` (see [Upstream changes](#upstream-changes-needed-in-ipm-drawio) below).
5. **Header injection** — every emitted `.impr.md` gets a warning header **inserted at the very top, before the source's H1** (one prepend, no source parsing). Standalone `.ipmt` files also get a wrapper `.impr.md` that embeds the SVG and carries the same header — every rendered artifact has a discoverable link home.
6. **Commit** — off by default. With `--commit`, ipm-render stages the changes in `ipm-intro-rendered` and creates one commit referencing the source SHA. Intended for CI; off locally so the user can inspect the diff first.

## Header format

Injected at the top of every `.impr.md`:

```markdown
> **Auto-generated by [ipm-render](https://github.com/infinite-pm/ipm-intro/tree/main/tools/ipm-render) — do not edit.**
> Source: [`docs/examples/tshirt-magic.md`](../../../ipm-intro/docs/examples/tshirt-magic.md) · [view on GitHub](https://github.com/infinite-pm/ipm-intro/blob/main/docs/examples/tshirt-magic.md)
> Rendered from commit [`<sha>`](https://github.com/infinite-pm/ipm-intro/commit/<sha>).

---
```

- **Source-link prefix** — follows the convention in [loops-ipm-swe/kb/relative-links.md](https://github.com/infinite-pm/loops-ipm-swe/blob/main/kb/relative-links.md) (local: [../../../loops-ipm-swe/kb/relative-links.md](../../../loops-ipm-swe/kb/relative-links.md)). Two flags:
  - `--source-prefix` — path from **rendered-repo root** to **source-repo root**. Configurable, depth-invariant. Default for sibling checkouts: `../ipm-intro/`. May be absolute (`/abs/path/` or `file:///...`) — used verbatim, no depth correction.
  - `--source-prefix-mode` (optional) — `relative` (default, prepend `depth × "../"` per file) or `absolute` (skip depth correction).
  - Per-file link is built in three legs per the kb: `(depth × "../") + source-prefix + source-relative-path`. So a relative prefix of `../ipm-intro/` produces `../../ipm-intro/docs/some.md` for a file at depth 1, `../../../ipm-intro/docs/examples/foo.md` for depth 2, etc. — `ipm-render` does the arithmetic.
- **GitHub link** — `--source-github-base` (default `https://github.com/infinite-pm/ipm-intro`) + branch (default `main`) + source-relative path. Pure string concat (GitHub URLs have no depth concept).
- **Commit SHA** — source HEAD at render time, captured by `git -C <source-repo> rev-parse HEAD`.

## Inputs / outputs

| Source (`ipm-intro/docs/`) | Rendered (`ipm-intro-rendered/docs/`) |
| --- | --- |
| `examples/tshirt-magic.md` | `examples/tshirt-magic.impr.md` (header + inserted images) + `examples/tshirt-magic__NN.ipm.svg` (and `.drawio.svg` if enabled) |
| `examples/murder-full.ipmt` | `examples/murder-full.impr.md` (header + embedded SVG) + `examples/murder-full.ipm.svg` (and `.drawio.svg` if enabled) |

## Dependencies

- [ipm-drawio](../../../ipm-drawio/) — `ipmsvg-gen` (native SVG, pure-Go, accepts `.ipmt` directly); `ipmt-parse`, `drawio-gen`, `ipmt-inline-md` (draw.io SVG, needs Podman + renderer container).
- Sibling checkout of `ipm-intro-rendered` (path configurable, default `../../ipm-intro-rendered` relative to `ipm-intro`).

## Upstream changes needed in ipm-drawio

**Refactor — extract shared block-extraction package.** Both `cmd-dev/ipmt-inline-md` (draw.io output) and `cmd-dev/debug-layout-md` (layout debug output) independently scan `.md` files for ` ```ipmt ` blocks. Pulling the shared logic into something like `pkg/mdblocks/` lets `ipm-render` import it directly with no duplication. Suggested implementation order:

1. Extract block extraction (`scanIpmtBlocks`, name reservation, image-ref insertion) from `ipmt-inline-md` into `pkg/mdblocks/`.
2. Switch `ipmt-inline-md` to use the package (behavior unchanged; pure refactor).
3. Add `cmd/ipm-render/` that uses `pkg/mdblocks/` + `pkg/ipmsvg/` directly for the native path, and shells out to `ipmt-inline-md` (or calls `pkg/drawio/gen` directly) for the draw.io path.
4. Optional cleanup: migrate `debug-layout-md` to the shared package too.

No new flags on `ipmt-inline-md` are required — the refactor obsoletes the earlier "add `--mode native|drawio`" suggestion.

## Out of scope (for now)

- No use of `ipm-miner` (that pipeline mines repos/data into IPM graphs — different direction).
- No use of `ipm-server` (interactive canvas viewer — `ipm-render` produces static artifacts only).
- Both could plug in later: e.g. linking each rendered page to a live `ipm-server` URL, or having `ipm-miner` emit `.ipmt` snippets that feed back into `ipm-intro/docs/`.
